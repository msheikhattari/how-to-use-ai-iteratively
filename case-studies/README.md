## Why case studies matter

The method is universal, but it's easier to learn from specific applications than from abstract principles. Each case study shows the method working on a real problem — making it transferable to other real problems you face.

A reader scrolling case studies can think: "That's like my situation. Let me try what they did." That's the leverage.

## Contribute your own

1. Read [`_template.md`](./_template.md) for the format
2. Read [`../CONTRIBUTING.md`](../CONTRIBUTING.md) for how to submit
3. Write your case study and submit via pull request (or email the maintainer)

Doesn't need to be dramatic. A small win that compresses 2 weeks of work into 2 hours is just as teachable as a big one.

## What good case studies share

- Specific to a recognizable situation (not vague)
- Honest about the median answer the AI initially gave
- Clear about which patterns you used to push back
- Specific outcome with numbers/timelines where possible
- Lessons that transfer to others in similar situations

Read the existing case study to see the shape. Yours doesn't have to be that long — it has to be specific.
